# Raw Source Files

This folder contains the original Whisper transcription output and source audio for each talk. It is **not web-visible** — GitHub Pages serves from `docs/` only, so files here are version-controlled but not publicly accessible.

> ⚠ **Important:** Automated transcripts — including those generated by Whisper — are approximations. They may contain errors, omissions, or text that does not accurately reflect what was actually spoken. **The original audio or video recording is the de facto source of truth.** When in doubt, refer to the original media.

## Folder Structure

Each subfolder mirrors the corresponding event folder name in `docs/events/`:

```
raw/
  YYYYMMDD-talk-slug-venue-slug/
    whisper.txt    ← raw Whisper plain text output
    whisper.srt    ← Whisper subtitle file with timestamps
```

**Example:**
```
raw/
  20260209-koshas-escuela-de-yoga-satyam/
    whisper.txt
    whisper.srt
  20260210-living-fully-in-yourself-yoga-carrasco/
    whisper.txt
    whisper.srt
```

## Naming Convention

Folder names: `YYYYMMDD-talk-slug-venue-slug/`
- Date prefix enables chronological sorting
- Lowercase, hyphen-separated words
- Match the event folder name in `docs/events/` exactly

## Audio File Storage

Audio files are large and require special handling. **Git LFS (Large File Storage)** is the recommended approach — files appear in the repo at their normal paths but the binary content is stored in GitHub's LFS blob store rather than git history.

If LFS is enabled, audio files go here alongside the Whisper outputs:

```
raw/
  20260209-koshas-escuela-de-yoga-satyam/
    audio.m4a      ← source recording (stored via Git LFS)
    whisper.txt
    whisper.srt
```

Files tracked via LFS: `*.m4a`, `*.mp3`, `*.wav`, `*.aac`
Files tracked normally: `*.txt`, `*.srt`, `*.md`

Until LFS is set up, the primary source of truth for audio/video is YouTube. The YouTube URL for each talk is recorded in the transcript header and in `docs/events.json`.

## Future: Automated Transcription

When audio-to-text automation is added, the workflow becomes:
1. Place `audio.m4a` in `raw/FOLDER/`
2. Run Whisper locally or via CI to generate `whisper.txt` and `whisper.srt`
3. Feed outputs into `prompts/transcript-en.md` for transcript generation

## Usage

When generating a new transcript, provide the contents of `whisper.txt` (and optionally `whisper.srt`) as input to the prompt in `prompts/transcript-en.md`.

The raw Whisper file is also needed during Phase 2 (Spanish translation) to recover any original Spanish audio segments for RECONSTRUCTED blocks.
